# Writing Guidelines

## 标题层级规范

```markdown
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（小节）
#### 四级标题（子小节）
##### 五级标题（细节说明）
```

**规则**：
- 每个文档只有一个一级标题
- 二级标题作为主要章节划分
- 避免跳级（如从##直接到####）
- 标题使用简洁明确的描述

## 代码块格式要求

```markdown
\`\`\`language
// 代码内容
\`\`\`
```

**语言标识**：
| 语言 | 标识 |
|------|------|
| JavaScript/JSX | `javascript` 或 `jsx` |
| TypeScript | `typescript` 或 `ts` |
| Python | `python` 或 `py` |
| Bash | `bash` 或 `sh` |
| JSON | `json` |
| YAML | `yaml` |
| Markdown | `markdown` 或 `md` |

**代码注释**：
- 关键步骤添加中文注释
- 解释复杂逻辑
- 标注注意事项

## 图片引用规范

### 在线图片
```markdown
![图片描述](https://example.com/image.png)
```

### 带标题的图片
```markdown
![图片描述](https://example.com/image.png "图片标题")
```

### 图片注意事项
- 使用稳定的官方资源URL
- 提供简洁的描述文字
- 必要时添加图片标题

## 参考文献格式

### 网页资源
```markdown
- [资源标题](URL)
```

### 带作者的资源
```markdown
- 作者. [资源标题](URL). 发布时间
```

### 学术论文
```markdown
- 作者. 论文标题. 期刊/会议, 年份
```

## 文本格式规范

### 强调
```markdown
**粗体强调**
*斜体*
***粗斜体***
```

### 行内代码
```markdown
使用`代码`引用函数名、命令、配置项
```

### 链接
```markdown
[链接文字](URL)
[链接文字](URL "链接标题")
```

## 列表格式

### 无序列表
```markdown
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2
```

### 有序列表
```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 任务列表
```markdown
- [ ] 未完成任务
- [x] 已完成任务
```

## 表格格式

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
```

**对齐方式**：
```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容   | 内容 | 内容   |
```

## 引用格式

```markdown
> 引用内容
>
> 多行引用
```

## 分隔线

```markdown
---
```

## 最佳实践

1. **行宽**：建议每行不超过80-100字符
2. **空行**：段落之间空一行
3. **标点**：中文内容使用全角标点
4. **数字**：阿拉伯数字使用半角
5. **空格**：中英文之间添加空格
